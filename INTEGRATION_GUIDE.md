# Integration Guide: Adding Tree Label Converter to Your Repository

This guide shows how to add the generic tree label converter to your `protein-sequence-analysis` repository.

## 🔧 Setup Steps

### Step 1: Copy the Script

```bash
cp convert_tree_labels_generic.py protein-sequence-analysis/scripts/convert_tree_labels.py
```

### Step 2: Copy Configuration

```bash
cp ebolavirus_mapping_updated.json protein-sequence-analysis/config/ebolavirus_mapping.json
```

### Step 3: Update Your README

Add this section to your `README.md` in the "Visualizing Results" section:

```markdown
### Step 4: Convert Tree Labels for Publication

The phylogenetic tree SVG generated by IQ-TREE contains cryptic sequence IDs. Convert these to readable species names:

**Basic usage:**
```bash
docker compose run ebolavirus-analysis \
  python scripts/convert_tree_labels.py \
  results/tree.svg \
  -c config/ebolavirus_mapping.json
```

This creates: `results/tree_labeled.svg` with clear Ebolavirus subtype names.

**With legend (adds species color codes):**
```bash
docker compose run ebolavirus-analysis \
  python scripts/convert_tree_labels.py \
  results/tree.svg \
  -c config/ebolavirus_mapping.json \
  -l
```

**See all options:**
```bash
docker compose run ebolavirus-analysis \
  python scripts/convert_tree_labels.py --help
```

The tool is organism-independent. To use with other datasets, create a custom config:
- Copy: `config_template.json`
- Edit with your sequence IDs and species names
- Run: `python scripts/convert_tree_labels.py your_tree.svg -c your_config.json`
```

## 🎯 Usage in Your Workflow

### For Standard Ebolavirus Analysis

```bash
# After running pipeline.py, simply run:
docker compose run ebolavirus-analysis \
  python scripts/convert_tree_labels.py \
  results/tree.svg \
  -c config/ebolavirus_mapping.json \
  -l
```

### For Different Proteins

If analyzing nucleoprotein, polymerase, or VP40 instead of glycoprotein, the tool still works:

```bash
docker compose run ebolavirus-analysis \
  python scripts/convert_tree_labels.py \
  results/tree.svg \
  -c config/ebolavirus_mapping.json
```

The mapping works for any Ebolavirus protein because it maps UniProt IDs to species names.

### For Other Organisms

1. Create your mapping config:
```bash
cp config/ebolavirus_mapping.json config/my_organism.json
```

2. Edit `config/my_organism.json` with your sequence IDs

3. Run converter:
```bash
docker compose run ebolavirus-analysis \
  python scripts/convert_tree_labels.py \
  results/tree.svg \
  -c config/my_organism.json
```

## 📝 Configuration Files to Include

### `config/ebolavirus_mapping.json` (example)

```json
{
  "description": "Ebolavirus protein sequence mapping",
  "organism": "Ebolavirus (6 species)",
  "id_mapping": {
    "Q66814": "Zaire ebolavirus (EBOV)",
    "P60173": "Sudan ebolavirus (SUDV)",
    "R4P4N7": "Monogeneric Ebola-like virus",
    "Q66799": "Reston ebolavirus (RESTV)",
    "H2SMH4": "Sudan ebolavirus (SUDV)",
    "P18279": "Zaire ebolavirus (EBOV)",
    "B1AAA7": "Sudan ebolavirus (SUDV)"
  },
  "label_colors": {
    "Zaire ebolavirus (EBOV)": "#E74C3C",
    "Sudan ebolavirus (SUDV)": "#3498DB",
    "Reston ebolavirus (RESTV)": "#9B59B6",
    "Monogeneric Ebola-like virus": "#95A5A6"
  }
}
```

### `config/config_template.json` (for custom organisms)

Include this template so users can create their own mappings:

```json
{
  "description": "Template for creating your own organism mapping",
  "organism": "Your organism name here",
  "id_mapping": {
    "SEQUENCE_ID_1": "Species/Strain Name",
    "SEQUENCE_ID_2": "Another Species",
    "SEQUENCE_ID_3": "More Sequences"
  },
  "label_colors": {
    "Species/Strain Name": "#FF6B6B",
    "Another Species": "#4ECDC4",
    "More Sequences": "#95E1D3"
  }
}
```

## 🚀 Advanced Features for Your Pipeline

### Batch Processing Multiple Trees

```bash
# Process multiple tree files
for tree_file in results/*.svg; do
  python scripts/convert_tree_labels.py \
    "$tree_file" \
    -c config/ebolavirus_mapping.json \
    -l
done
```

### Automated Pipeline Integration

Add to your `pipeline.py` at the end:

```python
import subprocess
import json

# After tree building is complete
tree_svg = 'results/tree.svg'
output_svg = 'results/tree_publication_ready.svg'
config = 'config/ebolavirus_mapping.json'

# Convert labels
result = subprocess.run([
    'python', 'scripts/convert_tree_labels.py',
    tree_svg, '-o', output_svg, '-c', config, '-l', '-v'
], check=True)

print(f"✅ Publication-ready tree: {output_svg}")
```
